原型模式

发表于 2015-08-21   |   分类于 设计模式 ， 创建型设计模式   |  

原型模式（Prototype Pattern）是一种创建型设计模式。意图：用原型实例指定创建对象的种类，并且通过拷贝这些原型创建新的对象。

代码实现

对象类

[Serializable]
public class SalesDepartment
{
    public string _manager;
    public string _employee_A;
    public string _employee_B;

    public SalesDepartment(string manager, string employee1, string employee2)
    {
        _manager = manager;
        _employee_A = employee1;
        _employee_B = employee2;
    }
}

[Serializable]
public class FinanceDepartment
{
    public string _manager;
    public string _employee_C;
    public string _employee_D;

    public FinanceDepartment(string manager, string employee1, string employee2)
    {
        _manager = manager;
        _employee_C = employee1;
        _employee_D = employee2;
    }

}

[Serializable]
public class Company
{
    //总裁
    public string _ceo;
    //经理
    public string _manager;
    //销售部
    public SalesDepartment _salesDepartMent;
    //财务部
    public FinanceDepartment _finaceDepartMent;

    public Company(string ceo, string manager)
    {
        _ceo = ceo;
        _manager = manager;

        _salesDepartMent = new SalesDepartment("张三", "员工sales1", "员工sales1");
        _finaceDepartMent = new FinanceDepartment("李四", "员工finace", "员工finace1");
    }

    public Company clone(bool isDeepClone)
    {
        Company company = null;
        if (isDeepClone)
        {
            MemoryStream memoryStream = new MemoryStream();
            BinaryFormatter formatter = new BinaryFormatter();
            formatter.Serialize(memoryStream, this);
            memoryStream.Position = 0;
            company = (Company)formatter.Deserialize(memoryStream);
        }
        else
        {
            company = (Company)this.MemberwiseClone();
        }

        return company;
    }
}

测试部分代码

static void Main(string[] args)
{
    Company test1 = new Company("ceo", "manager");
    Company test2 = test1.clone(false);

    test1._finaceDepartMent._manager = "赵六";

    Console.WriteLine(test1._ceo.Equals(test2._ceo)? test1._ceo: (test1._ceo+ "" + test2._ceo));

    string manager1, manager2;
    manager1 = test1._finaceDepartMent._manager;
    manager2 = test1._finaceDepartMent._manager;
    Console.WriteLine(manager1.Equals(manager2) ? manager1 : (manager1 + "" + manager2));

    Company test3 = new Company("ceo3", "manager4");
    Company test4 = test3.clone(true);

    test3._finaceDepartMent._manager = "赵六";

    Console.WriteLine(test1._ceo.Equals(test2._ceo) ? test1._ceo : (test1._ceo + "" + test2._ceo));

    manager1 = test3._finaceDepartMent._manager;
    manager2 = test4._finaceDepartMent._manager;
    Console.WriteLine(manager1.Equals(manager2) ? manager1 : (manager1 + "" + manager2));

    Console.Read();
}

prototype原型要实现一个clone的方法来拷贝自身对象。

在Company类中，实现了一个深拷贝和一个浅拷贝。通过测试我们发现深拷贝的时候，当财务经理变化的时候，拷贝类的财务经理并没有变化。而浅拷贝的类在原类的财务经理变化的时候，拷贝类的财务经理发生了变化。我们可以理解为，浅拷贝只是给原来的公司换了个名字，深拷贝是按照原来的公司的结构创建了另外的一家相同结构的公司。

个人理解

原型是一个对原类拷贝，但是它并不调用构造函数。比起用new来创建对象，在效率上会提高。但是原型模式会出现深拷贝和浅拷贝的问题，对于基本类型都是值拷贝，对于数组，容器，对象等都是引用拷贝，如果要实现值拷贝，就要自己new出各个类的手工实例或者通过序列化和反序列化来实现拷贝。

适用性：

当要实例化的类是在运行时刻指定时，例如，通过动态装载。

为了避免创建一个与产品类层次平行的工厂类层次时。

当一个类的实例只能有几个不同状态组合中的一种时。建立相应数目的原型并克隆它们可能比每次用合适的状态手工实例化该类更方便一些。

如果你创建的对象是多变化、多等级的产品，或者产品的创建过程非常耗时的时候(比如，有一定的计算量，或者对象创建时需要从网络或数据库中获取一定的数据)，或者想把产品的创建独立出去，不想了解产品创建细节的时候可以考虑使用。不得不说，原型模式给了我们多一种创建对象，并且不依赖具体对象的选择。

特点：

对客户端隐藏了具体的产品类。

允许客户只通过注册原型实例就可以将一个具体产品类并入到系统中，客户可以在运行时刻建立和删除原型；

减少了子类的构造。原型模式是Clone一个原型而不是请求工厂方法创建一个，所以它不需要一个与具体产品类平行的Creator类层次。

原型模式具有给一个应用软件动态加载新功能的能力。由于Prototype的独立性较高，可以很容易动态加载新功能而不影响旧系统；

产品类不需要非得有任何事先确定的等级结构，因为原型模式适用于任何的等级结构。

原型模式的最重要缺点就是每一个类必须配备一个Clone方法，而且这个Clone方法需要对类的功能进行通盘考虑。这对全新的类来说不是很难，但对已有的类进行改造时，不一定是容易的事。

建造者模式

发表于 2015-08-20   |   分类于 设计模式 ， 创建型设计模式   |  

建造者模式(BuilderPattern)是一种创建型设计模式。意图：将一个复杂对象的构建与它的表示分离，使得同样的构建过程可以创建不同的表示。

产品类

class Product
{
    private string _partHead { get; set; }
    private string _partBody { get; set; }
    private string _partFoot { get; set; }

    public void setHead(string property)
    {
        _partHead = property;
    }

    public void setBody(string property)
    {
        _partBody = property;
    }

    public void setFoot(string property)
    {
        _partFoot = property;
    }


    public void show() 
    {
        Console.WriteLine("Head :" + _partHead);
        Console.WriteLine("Body :" + _partBody);
        Console.WriteLine("Foot :" + _partFoot);
    }
}

建造类

abstract class Builder
{        
    public abstract void BuildHead();

    public abstract void BuildBody();

    public abstract void BuildFoot();

    public abstract Product GetProduct();
}

class ConcretBuilderAnimal : Builder
{
    private Product _product;

    public ConcretBuilderAnimal()
    {
        _product = new Product();

    }
    public override void BuildHead()
    {
        _product.setHead("this is animal head!");
    }

    public override void BuildBody()
    {
        _product.setBody("this is animal body!");
    }

    public override void BuildFoot()
    {
        _product.setFoot("this is animal foot!");
    }

    public override Product GetProduct()
    {
        return _product;
    }
}

class ConcretBuilderMan : Builder
{
    private Product _product;

    public ConcretBuilderMan()
    {
        _product = new Product();
    }

    public override void BuildHead()
    {
        _product.setHead("this is man head!");
    }

    public override void BuildBody()
    {
        _product.setBody("this is man body!");
    }

    public override void BuildFoot()
    {
        _product.setFoot("this is man foot!");
    }

    public override Product GetProduct()
    {
        return _product;
    }
}

导演类

class Director
{
    public void Construct(Builder builder)
    {
        builder.BuildHead();
        builder.BuildBody();
        builder.BuildFoot();
    }
}

具体测试和实现

class Program
{
    static void Main(string[] args)
    {
        Builder builder = new ConcretBuilderMan();
        Director director = new Director();
        director.Construct(builder);

        Product product = builder.GetProduct();
        product.show();

        Console.ReadLine();
    }
}

建造类负责构造对象的细节，由导演类统一的规范流程来构造对象。使得同样的流程可以建造出不同的对象。

个人理解

举汽车建造过程的例子来说明：建造者生产不同汽车的轮胎，外壳， 系统。然后将这些产品拿到外面的工厂按照统一的组装成汽车。但是出来的汽车可能是甲壳虫，奥迪等。该模式的特点是：建造对象的算法，既独立于对象的组成部分，也独立于对象的装配过程。工厂模式中， 对象装配过程也在对象中。两者的使用区别是：构造复杂的对象，而且客户端不必知道产品的细节的时候，用工厂模式，构造更加复杂的对象用建造者模式

好处：

客户端不必知道产品的细节。

建造者相互独立，有利于系统扩展。

建造者是独立的，可以对建造的过程进行细化，而不影响其他的模块。

适用范围：

构建复杂的对象，构建的流程是统一的，构建的组件是复杂的。

构建对象的算法，既独立于构建过程，也独立于组成部分的装配方法。

单例模式

发表于 2015-08-19   |   分类于 设计模式 ， 创建型设计模式   |  

单例模式(Singleton Pattern)是一种创建型设计模式。意图：保证一个类仅有一个实例，并提供一个访问它的全局访问点。

懒汉式

public class SingletonA
{
    private static SingletonA _singleton;
    private SingletonA()
    {
    }

    public static SingletonA GetInstance()
    {
        if (null == _singleton)
        {
            _singleton = new SingletonA();
        }

        return _singleton;
    }
}

static void Main(string[] args)
{
    SingletonA a1 = SingletonA.GetInstance();
    SingletonA a2 = SingletonA.GetInstance();
    if (a1.Equals(a2))
    {
        Console.WriteLine("a1 = a2");
    }
    else
    {
        Console.WriteLine("a1 != a2");
    }

    Console.ReadLine();
}

饿汉式

public class SingletonB
{
    private static SingletonB _singleton = new SingletonB();
    private SingletonB()
    {
    }

    public static SingletonB GetInstance()
    {
        return _singleton;
    }
}

static void Main(string[] args)
{
    SingletonB b1 = SingletonB.GetInstance();
    SingletonB b2 = SingletonB.GetInstance();
    if (b1.Equals(b2))
    {
        Console.WriteLine("b1 = b2");
    }
    else
    {
        Console.WriteLine("b1 != b2");
    }

    Console.ReadLine();
}

双重检查锁定

public class SingletonLock
{
    private static SingletonLock _singleton;
    private static readonly Object sysncObject = new Object();

    private SingletonLock()
    {
    }

    public static SingletonLock GetInstance()
    {
        if (null == _singleton)
        {
            lock (sysncObject)
            {
                if(null == _singleton) {
                    _singleton = new SingletonLock();
                }
            }
        }
        return _singleton;
    }

}

static void Main(string[] args)
{    
    SingletonLock lock1 = SingletonLock.GetInstance();
    SingletonLock lock2 = SingletonLock.GetInstance();
    if (lock1.Equals(lock2))
    {
        Console.WriteLine("lock1 = lock2");
    }
    else
    {
        Console.WriteLine("lock1 != lock2");
    }

    Console.ReadLine();
}

适用性

适用性 整个应用程序的生命周期只需要一个实例。

各种写法的比较 懒汉式适用于单线程的程序，在多线程的程序中会造成多个实例。 双重锁检查的写法可以很好的解决多线程的问题。饿汉式的写法在类装载的时候就出事话，没有达到懒汉式装载的效果。

工厂模式

发表于 2015-08-19   |   分类于 设计模式 ， 创建型设计模式   |  

工厂模式(Factory Pattern)是一种创建型设计模式。意图：定义一个用于创建对象的接口，让子类决定实例化哪一个类。Factory Method 使一个类的实例化延迟到其子类。

简单工厂模式

public class Product
{
    public virtual void show()
    {
    }

    public string className()
    {
        return this.GetType().Name;
    }
}

public class ProductA : Product
{
    public override void show()
    {
        Console.WriteLine("this is " + className());
    }
}

public class ProductB : Product
{
    public override void show()
    {
        Console.WriteLine("this is " + className());
    }
}

enum ProductType {
    A = 0,
    B,
    UNKONW= -1
}

class SimpleFactory
{
    public Product createProduct(ProductType type)
    {
        Product product = null;
        switch (type)
        {
        case ProductType.A:
            product = new ProductA();
            break;
        case ProductType.B:
            product = new ProductB();
            break;
        }

        return product;
    }
}

static void Main(string[] args)
{
    SimpleFactory factory = new SimpleFactory();

    Product product = factory.createProduct(ProductType.A);
    product.show();

    Console.ReadLine();
}

工厂模式

public class Factory
{
    public virtual Product createProduct()
    {
        return null;
    }
}

public class FactoryA : Factory
{
    public override Product createProduct()
    {
        return new ProductA();
    }
}

public class FactoryB : Factory
{
    public override Product createProduct()
    {
        return new ProductB();
    }
}

static void Main(string[] args)
{
    Factory factory = new FactoryB();
    Product product = factory.createProduct();
    product.show();
}

抽象工厂模式

public class AbstractFactory
{
    public virtual Product createProductA()
    {
        return null;
    }

    public virtual Product createProductB()
    {
        return null;
    }
}

public class AbstractFactoryA : AbstractFactory
{
    public override Product createProductA()
    {
        return new ProductA1();
    }

    public override Product createProductB()
    {
        return new ProductB1();
    }
}

public class AbstractFactoryB : AbstractFactory
{
    public override Product createProductA()
    {
        return new ProductA();
    }

    public override Product createProductB()
    {
        return new ProductB();
    }

}

public class ProductA1 : Product
{
    public override void show()
    {
        Console.WriteLine("this is " + className());
    }
}

public class ProductB1 : Product
{
    public override void show()
    {
        Console.WriteLine("this is " + className());
    }
}

static void Main(string[] args)
{
    AbstractFactory factory = new AbstractFactoryA();
    Product product = factory.createProductA();
    product.show();
}

优缺点

适用性：类的实例由子类指定。

优缺点：简单工厂模式实现了责任的分离，也造成了工厂类的复杂性，每添加一个产品都必须修改工厂类，不符合开放和封闭原则。工厂模式
解决了简单工厂模式工厂类的复杂性问题，但是产品多样的时候，工厂类众多。抽象工厂模式实现了一类产品的构造。
