访问者模式

发表于 2015-09-23   |   分类于 设计模式 ， 行为型设计模式   |  

访问者模式(VisitorPattern)是一种行为型设计模式。意图：封装某些作用于某种数据结构中各元素的操作，它可以在不改变数据结构的前提下定义作用于这些元素的新的操作。

代码实现

抽象访问者

抽象访问者（Visitor）:抽象类或者接口，具体到程序中就是接口中定义哪些对象是可以访问的。

interface IVisitor
{
    void Visit(ConcreteElementA element);
    void Visit(ConcreteElementB element);
}

访问者

访问者（ConcreteVisitor）：实现抽象访问者定义的接口，并确定访问一个类要做什么操作。

class ConcreteVisitor : IVisitor
{
    public void Visit(ConcreteElementA element)
    {
        element.Dosomething();
    }

    public void Visit(ConcreteElementB element)
    {
        element.Dosomething();
    }
}

抽象元素类

抽象元素类（Element）:声明接收哪一类的访问者的操作，并定义自身的逻辑。

abstract class Element
{
    public abstract void Accept(IVisitor visitor);
    public abstract void Dosomething();
}

元素类

元素类（ConcreteElement）：实现抽象元素类的方法和接口。

class ConcreteElementA : Element
{
    public override void Accept(IVisitor visitor)
    {
        visitor.Visit(this);
    }

    public override void Dosomething()
    {
        Console.WriteLine("this is element A");
    }
}

class ConcreteElementB : Element
{
    public override void Accept(IVisitor visitor)
    {
        visitor.Visit(this);
    }

    public override void Dosomething()
    {
        Console.WriteLine("this is element B");
    }
}

结构对象

结构对象（ObjectStruture ）：一个元素集合。

class ObjectStucture
{
private IList _elements = new List();

    public ObjectStucture()
    {
        for (int i = 0; i < 10; i++)
        {
            if (1 == i % 2)
            {
                _elements.Add(new ConcreteElementA());
            }
            else
            {
                _elements.Add(new ConcreteElementB());
            }
        }
    }

    public IList<Element> GetElements()
    {
        return _elements;
    }
}

测试代码

class Program
{
    static void Main(string[] args)
    {
        ObjectStucture objs = new ObjectStucture();
        foreach (Element elemnt in objs.GetElements())
        {
            elemnt.Accept(new ConcreteVisitor());
        }

        Console.Read();
    }
}

个人理解

适用性

一个对象中存在与本身不相关（或关系弱）的操作，为了避免这些操作污染对象，可以通过一个访问者来吧这类操作封装起来。

一组对象中存在相似的操作，为了避免出现大量重复的操作，可以把这些操作封装到访问者中。
