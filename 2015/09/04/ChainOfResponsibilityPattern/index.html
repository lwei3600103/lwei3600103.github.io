责任链模式

发表于 2015-09-04   |   分类于 设计模式 ， 行为型设计模式   |  

责任链模式(ChainOfResponsibilityPattern)是一种行为型设计模式。意图：使多个对象都有机会处理请求，从而避免发送者和接收者之间的耦合关系。将所有可以处理请求的对象串成一条链，并沿着这条链传递请求，直到有对象处理它。

代码实现

请求对象

class Request
{
    private int _day;
    private string _reason;


    public int Day
    {
        get { return _day; }
        set { _day = value; }
    }

    public string Reason 
    {
        get { return _reason; }
        set { _reason = value; }
    }
}

一个抽象“链”对象

抽象对象定义抽象的处理请求的方法，并定义下个链对象。

abstract class AbstractHandler
{
    private AbstractHandler _successor;
    protected string _name;

    public AbstractHandler(string name)
    {
        _name = name;
    }

    public AbstractHandler Successor 
    {
        get { return _successor; }
        set { _successor = value; }
    }

    public abstract bool PassHandle(Request request);
}

具体的请求处理对象

具体的对象根据条件实现请求的方法。

class PM : AbstractHandler
{
    public PM(string name) :
        base(name)
    {

    }

    public override bool PassHandle(Request request)
    {
        if (request.Day < 0.5)
        {
            Console.WriteLine("PM " + _name + " 批准请假。 ");
            return true;
        }

        return Successor.PassHandle(request);
    }
}

class HR : AbstractHandler
{
    public HR(string name) :
        base(name)
    {

    }

    public override bool PassHandle(Request request)
    {
        if (request.Day > 0.5 && request.Day < 3) 
        {
            Console.WriteLine("HR " + _name + " 批准请假。 ");
            return true;
        }

        return Successor.PassHandle(request);
    }
}

class Manager : AbstractHandler
{
    public Manager(string name) :
        base(name)
    {

    }

    public override bool PassHandle(Request request)
    {
        if (request.Reason.Equals("正当理由"))
        {
            Console.WriteLine("Manager " + _name + " 批准请假。 ");
            return true;
        }

        Console.WriteLine("Manager " + _name + " 认为理由不正当，不允许批准。 ");
        return false;
    }
}

测试代码

class Program
{
    static void Main(string[] args)
    {
        PM pm = new PM("项目经理");
        HR hr = new HR("人力资源");
        Manager manager = new Manager("总经理");

        pm.Successor = hr;
        hr.Successor = manager;


        Request request1 = new Request();
        request1.Day = 2;
        request1.Reason = "正当理由";

        pm.PassHandle(request1);

        Request request2 = new Request();
        request2.Day = 6;
        request2.Reason = "不正当理由";

        pm.PassHandle(request2);

        Console.Read();
    }
}

个人理解

责任链模式通过一条链来处理请求，请求者并不知道处理者对象，分离了请求者和处理者。但是每个请求都必须从链头开始处理，造成了内存中的垃圾对象。

适用性

有多个对象处理请求时，而且请求在运行时候决定。

在不明确处理者的时候，向多个对象提出一个请求。

可处理一个请求的对象集合应该被动态指定。

如果存在N对N，或者是常规线性关系，我们完全可以用表驱动代替责任链。

责任链并非只可以在上下级间传递，也可以在同级间传递。
