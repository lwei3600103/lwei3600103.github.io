代理模式

发表于 2015-08-30   |   分类于 设计模式 ， 创建型设计模式   |  

代理模式(ProxyPattern)是一种创建型设计模式。意图：为其他对象提供一种代理以控制对这个对象的访问。

代码实现

普通代理

接口类

提供公有的方法

public interface IGamePlayer
{
    void Login(string user, string password);

    void KillBoss();

    void Update();

    IGamePlayer GetProxy();
}
实例类

完成接口的方法

public class GamePlayer : IGamePlayer
{
    private string _user;

    public void Login(string user, string password)
    {
        _user = user;
        Console.WriteLine("user is " + user + " password is " + password);
    }

    public void KillBoss()
    {
        Console.WriteLine(_user + " kill a boss.");
    }

    public void Update()
    {
        Console.WriteLine(_user + " update one. ");
    }

    public IGamePlayer GetProxy()
    {
        return null;
    }
}
代理类
public class Proxy : IGamePlayer
{
    IGamePlayer _player;

    public Proxy(IGamePlayer player)
    {
        _player = player;
    }

    public void Login(string user, string password)
    {
        _player.Login(user, password);
    }

    public void KillBoss()
    {
        _player.KillBoss();
    }

    public void Update()
    {
        _player.Update();
    }

    public IGamePlayer GetProxy()
    {
        return this;
    }
}
测试代码
class Program
{
    static void Main(string[] args)
    {
        IGamePlayer player = new GamePlayer();
        IGamePlayer proxy = new Proxy(player);

        proxy.Login("test", "****");
        proxy.KillBoss();
        proxy.Update();

        Console.Read();
    }
}

强制代理

一般都是通过代理找到真实对象，强制代理却通过真实对象找到代理。

实体类

强制代理 只需要将实例类中的方法加入获得代理。 代理必须是真实对象，不能是外部new出来的。

public class ForceGamePlayer : IGamePlayer
{
    private string _user;
    private IGamePlayer _proxy = null;

    public IGamePlayer GetProxy()
    {
        _proxy = new Proxy(this);
        return _proxy;
    }

    public void Login(string user, string password)
    {
        _user = user;

        if (IsProxy())
        {
            Console.WriteLine("user is " + user + " password is " + password);
        }
        else
        {
            Console.WriteLine(" please use proxy. ");
        }

    }

    public void KillBoss()
    {
        if (IsProxy())
        {
            Console.WriteLine(_user + " kill a boss.");
        }
        else
        {
            Console.WriteLine(" please use proxy. ");
        }

    }

    public void Update()
    {
        if (IsProxy())
        {
            Console.WriteLine(_user + " update one. ");
        }
        else
        {
            Console.WriteLine(" please use proxy. ");
        }

    }

    public bool IsProxy()
    {
        if (null == _proxy)
        {
            return false;
        }
        else
        {
            return true;
        }
    }
}
测试代码
class Program
{
    static void Main(string[] args)
    {
        IGamePlayer forcePlayer = new ForceGamePlayer();
        IGamePlayer forceProxy = forcePlayer.GetProxy();
        forceProxy.Login("forcePlayer", "****");
        forceProxy.KillBoss();
        forceProxy.Update();

        Console.Read();
    }
}

有个性的代理

代理不仅可以完成主体接口，也可以实现其他接口不能完成的任务。

实体类

实体类可以继承多个接口，实现个性化的接口。

public class PersonalGamePlayerProxy : IGamePlayer, IProxy
{
    IGamePlayer _player;

    public PersonalGamePlayerProxy(IGamePlayer player)
    {
        _player = player;
    }

    public IGamePlayer GetProxy()
    {
        return this;
    }

    public void Login(string user, string password)
    {
        _player.Login(user, password);
    }

    public void KillBoss()
    {
        _player.KillBoss();
    }

    public void Update()
    {
        Money();
        _player.Update();
    }

    public void Money()
    {
        Console.WriteLine(" update on need $15. ");
    }
}
新增加的接口类
interface IProxy
{
    void Money();
}
测试代码
class Program
{
    static void Main(string[] args)
    {
        IGamePlayer personalPlayer = new GamePlayer();
        IGamePlayer personalProxy = new PersonalGamePlayerProxy(personalPlayer);
        personalProxy.Login("personalProxy", "****");
        personalProxy.KillBoss();
        personalProxy.Update();

        Console.Read();
    }
}

虚拟代理

在需要的时候才判断是否创建对象，可以避免代理对象较多引起的初始化缓慢的问题，缺点是要在每个方法中判断时候对象是否为null。

公用接口类
public interface Subject
{
    void Request();
}
实例类
public class RealSubject : Subject
{
    public void Request()
    {
        Console.WriteLine(" real subject ");
    }
}
虚拟代理类
public class VirtualProxy : Subject
{
    private Subject _subject;

    public void Request()
    {
        if (null == _subject)
        {
            _subject = new RealSubject();
        }

        _subject.Request();
    }
}
测试代码
class Program
{
    static void Main(string[] args)
    {
        VirtualProxy virtualProxy = new VirtualProxy();
        virtualProxy.Request();

        Console.Read();
    }
}

个人理解

在代理模式中，真实的角色实现业务逻辑，而不关心事务，事务由代理对象来处理。真实对象，只要实现了接口，不管怎么变化，代理类都不需要变化。

适用性

在要用比较通用和复杂的指针代替简单的指针的时候。

proxy并不一定要求保持接口的一致性，只要能够实现间接控制，有时候损及一些透明性是可以接受的。

“增加一层间接层”是软件系统中对许多负责问题的一种常见解决方法。

适配器Adapter 为它所适配的对象提供了一个不同的接口。相反，代理提供了与它的实体相同的接口。

尽管Decorator的实现部分与代理相似，但Decorator的目的不一样。Decorator为对象添加一个或多个功能，而代理则控制对对象的访问。
